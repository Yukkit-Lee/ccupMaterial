# 智检云盾项目导师汇报逐字稿（简明版）

> 建议时长：约 4—5 分钟

老师您好。我想汇报一下我准备参加挑战杯的项目，名称是“智检云盾——面向装备制造的多工厂数据安全协同质检平台”。这个项目的核心不是做一个普通的缺陷检测模型，而是研究多家工厂之间怎样在不交换原始质检数据的前提下，共同训练一个缺陷识别模型。

我先说明项目要解决的具体问题。

在装备制造企业里，视觉质检常用于识别零件表面的划伤、裂纹、凹坑、焊缝异常等缺陷。但单个工厂的数据往往不够用。正常件很多，裂纹等严重缺陷很少；而且每家工厂的相机、光照、材料、工艺和标注标准不同，所以一家的模型拿到另一家工厂后，效果可能明显下降。

比较直接的解决方法是把多家工厂的数据放在一起训练，但这在实际中很难做。质检图像里可能包含零件结构、工艺、批次和供应商信息，企业不愿意把原始图像和工艺数据交出去。因此，项目要解决的是：**数据不能集中时，怎样利用多家工厂的数据提高缺陷识别能力，同时尽量保护每家的数据隐私。**

本项目的研究主要分成两个部分。

第一部分是多工厂安全协同训练。设想有三方参与：零部件厂 A、零部件厂 B 和独立检测机构。A 和 B 各自保存自己的质检图像和缺陷标签，检测机构负责统一部分标签和验收规则。三方不把原始图片互相传输，而是在各自本地处理数据，只对联合训练必须使用的计算结果进行安全协同。项目采用的是秘密共享式的安全多方学习思路，不是常见的联邦学习路线。

首期原型不会一开始做得很大。我们先只做三方、一个二维表面缺陷分类任务、四个类别：正常、划伤、裂纹和凹坑。模型先用逻辑回归或小型 MLP。这样做的目的，是先把安全协同训练的正确性、训练时间和通信开销测清楚，而不是一开始就追求复杂模型。

第二部分是 DP-SGD，也就是差分隐私随机梯度下降。它解决的是另一个问题：即使原始数据不离开工厂，训练出来的模型也可能泄露信息。比如攻击者反复查询模型，可能推断某一张罕见缺陷图片是否参与过训练。DP-SGD 的作用是限制单条训练数据对最终模型参数的影响，从而降低这种成员推断风险。

DP-SGD 的具体做法可以概括为三步。第一步，对一个小批量中的每条样本分别计算梯度；第二步，给每条梯度设置上限，超过上限就裁剪，避免某一条样本对模型更新影响过大；第三步，在汇总后的梯度中加入随机高斯噪声，再更新模型参数。噪声大小、梯度裁剪阈值和训练轮数共同决定隐私保护强度，通常用隐私预算 ε 和失败概率 δ 来表示。

我们在项目中不是只说“加噪声就安全”，而是要测量隐私、精度和效率的关系。噪声太小，隐私保护不足；噪声太大，模型识别裂纹等缺陷的效果会下降。因此后续实验会对比三种情况：单厂训练、把数据明文集中训练、安全协同加 DP-SGD 训练。比较的指标包括准确率、Macro-F1、裂纹的漏报率、训练时延、通信量，以及隐私预算。这样才能回答这个方案在什么条件下值得使用。

在应用场景上，第一步准备放在长春汽车零部件的表面质检场景。例如两个零部件厂分别有冲压件、焊点或表面件的缺陷图片，但各自缺少足够的裂纹样本。通过协同训练，工厂 A 可以利用工厂 B 中的缺陷分布信息来提升模型，而不需要拿到 B 的原始图片。独立检测机构可以帮助统一标签，例如什么算裂纹、什么算划伤，以及什么指标可以作为验收标准。训练出的模型回到各自工厂本地使用，只作为质检人员筛查和复检的辅助工具，不代替人工做最终质量判定。

如果这个场景验证可行，后续可沿用同一套方法到其他东北制造场景。例如沈阳的机床、泵件、叶轮和铸件表面缺陷；哈尔滨的汽轮机叶片和锅炉焊缝；大连轨道交通装备的车体焊缝和机车部件外观缺陷。这些场景的共同点是：缺陷样本少，跨企业数据有价值，但原始数据又不能随意共享。

目前项目处于方案设计和原型构建阶段。下一步工作是先基于公开或授权数据搭建三方模拟环境，完成安全协同训练和 DP-SGD 的实验；再通过行业访谈确定真实企业的标签标准、数据边界和验收需求。我的研究重点就是把这个隐私保护机器学习方法在工业质检场景中做成一个可验证、可测量的原型。

## DP-SGD 算法说明（可用于老师追问）

### 1. 它解决什么问题？

普通 SGD 每次根据一个小批量数据的平均梯度更新参数。若某个训练样本很特殊，特别是少见的缺陷图像，它可能会对参数留下较明显的影响。攻击者有时可以利用模型输出或参数变化，推断该样本是否参加过训练。DP-SGD 的目标是降低单个样本对模型结果的可识别影响。

### 2. 每一次参数更新做什么？

设当前模型参数为 $\theta$，小批量为 $B$，单个样本 $x_i$ 的损失函数为 $L(\theta, x_i)$。

1. 分别计算每个样本的梯度：$g_i=\nabla_\theta L(\theta,x_i)$；
2. 设定裁剪阈值 $C$，将每个梯度裁剪为：
   $\bar{g}_i=g_i\cdot\min(1,C/\lVert g_i\rVert_2)$；
3. 对裁剪后的梯度求平均，并加入高斯噪声：
   $\tilde{g}=\frac{1}{|B|}\left(\sum_{i\in B}\bar{g}_i+\mathcal{N}(0,\sigma^2C^2I)\right)$；
4. 用 $\theta\leftarrow\theta-\eta\tilde{g}$ 更新模型，其中 $\eta$ 是学习率。

直观理解：裁剪先限制任何一条数据“能说多大声”，加噪声再让外部观察者难以分辨某一条数据是否真的参与过训练。

### 3. ε、δ、C 和 σ 分别是什么意思？

- $\varepsilon$：隐私预算。一般来说越小，隐私保护越强，但训练难度也越大；
- $\delta$：极小的允许失败概率，通常取远小于样本规模倒数的值；
- $C$：梯度裁剪阈值，决定单条样本梯度的最大影响；
- $\sigma$：噪声乘子，越大表示加入的噪声越多、隐私一般越强，但精度可能下降。

多轮训练后，不能只看某一步加入了多少噪声，还要通过隐私会计方法累计计算最终的 $(\varepsilon,\delta)$。因此，项目实验中必须同时报告模型效果、训练轮数、裁剪阈值、噪声乘子和最终隐私预算，不能只写“使用了差分隐私”。

### 4. 它和安全多方计算分别负责什么？

两者解决的问题不同，不能互相替代。

- 安全多方计算解决“训练过程中各方怎样协同计算而不直接看到对方原始数据”；
- DP-SGD 解决“最终模型会不会过度记住某一条训练样本”。

因此，本项目的逻辑是：用安全多方学习减少跨厂训练过程中的数据暴露，再用 DP-SGD 降低模型发布后的成员推断风险。

### 5. 项目中所谓“改进”应该怎么表述？

在没有完成对比实验前，不能笼统说算法已经显著优于已有方法。更准确的表述是：**针对小规模、多方、轻量质检模型的场景，研究梯度裁剪阈值、噪声乘子、批量大小和训练轮数的联合配置，并比较不同配置下的隐私预算、缺陷识别效果、训练时延和通信开销。** 只有完成可复现实验后，才能具体说明改进点和效果。
