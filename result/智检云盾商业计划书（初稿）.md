# 一、执行摘要

装备制造业的视觉质检正在加快智能化，但高价值零部件的缺陷数据分散于整机厂、零部件厂和检测机构。单一工厂往往缺少罕见缺陷样本，难以训练出稳定的识别模型；而质检图像、工艺参数、批次信息又关联企业工艺秘密和供应链责任，难以直接汇集。智检云盾面向这一矛盾，提供“数据不出厂、模型可协同”的多工厂数据安全协同质检平台。

平台以本地部署为基础：企业在本地完成零件、焊缝、涂装等质检图像的特征提取；各方仅在秘密共享协议下参与联合训练，原始图像与工艺数据不向其他参与方公开；对模型训练采用差分隐私机制，降低模型发布后被成员推断攻击的风险。平台首先服务于汽车零部件表面缺陷识别，可向重大装备、轨道交通、电站装备等场景延展。

项目技术路线参考 Ruan 等人在 IEEE S&P 2023 公开发表的 PEA（Private, Efficient, and Accurate）研究：使用安全多方计算支持差分隐私随机梯度下降，并以公开、数据无关的特征提取与浅层模型训练降低安全计算开销。团队拟在此基础上完成面向工业质检的工程化适配、原型验证、部署工具和数据治理流程设计，不将公开论文表述为团队自主成果。

项目拟采用“试点实施费 + 平台订阅费 + 运维与模型迭代服务费”的模式。近期以长春汽车零部件产业链为首期验证场景，构建“零部件厂 A—零部件厂 B—检测机构 C”三方模拟原型；中期复制至沈阳重大装备、哈尔滨电站装备和大连轨道交通装备产业链。项目现有登记经营主体为永清县臻青奇百货店（个体工商户），其经营范围、经营状态和与本项目的承载关系须以营业执照及实际情况为准；正式开展软件服务、签订大额合同或引入股权融资前，拟根据业务需要完成经营范围核验及项目主体调整。本计划书不主张该主体已产生本项目营收。

# 二、项目概述

## 1. 项目背景

东北拥有汽车、重大装备、能源电力装备、轨道交通装备和机床等传统产业基础。以长春汽车产业链、沈阳重大装备、哈尔滨电站装备、大连轨道交通装备为代表的制造场景，具有零件型号多、质量要求高、供应链协同深的特点。机器视觉质检能够减少人工复检压力、提高一致性，但模型质量依赖多样且高质量的缺陷数据。对单个企业而言，罕见缺陷数据不足；对产业链而言，直接集中质检图像、加工参数和供应商批次信息，又可能暴露工艺秘密、客户信息和质量责任边界。

数据安全与智能制造的双重需求，使“数据可用不可见”成为协同质检的关键。本项目契合专项赛“数智赋能产业”组别关于装备制造等传统优势产业智能化、绿色化、融合化发展的导向：以安全协同算法提升产业链质检效率，以标准化部署工具降低中小供应商使用门槛。

## 2. 项目定位与当前阶段

智检云盾定位为服务装备制造产业链的安全协同质检软件与技术服务平台。首期聚焦表面缺陷识别：正常、划伤、裂纹、凹坑、气孔、涂装异常等；后续再根据数据条件延展至焊缝、铸造件、叶轮和机加工件。

截至本计划书编制日，项目处于方案设计与原型构建规划阶段。尚未提供真实工厂数据、客户试点合同、第三方测评或已验证效果，因此本计划书不作“已落地”“已签约”“准确率达到某数值”等表述。原型阶段将使用公开或经授权的数据集、模拟多方节点完成可复现实验。

## 3. 项目应用

典型应用由三类主体组成：整机厂或核心制造企业、零部件供应商、第三方检测/质量服务机构。每一主体在本地保留其质检图像与标签；平台在不收集原始数据的前提下，训练共同使用的缺陷识别模型。模型部署回各主体本地，支持新增样本检测、缺陷风险提示、人工复检队列生成和版本审计。

首期拟验证场景为长春汽车零部件的表面缺陷协同识别。选择汽车零部件的原因在于图像任务边界清晰、产业链多主体关系明确，且与公开 PEA 研究已覆盖的图像分类范式较为接近。沈阳压缩机与泵、哈尔滨电站装备、大连轨道交通装备、沈阳机床可作为后续行业拓展对象，不代表项目已与相关企业合作。

## 4. 社会价值

项目通过减少数据孤岛造成的模型偏差，提高罕见缺陷的识别机会，帮助制造企业降低重复标注和重复建模成本；通过本地部署、最小必要数据使用、训练记录审计等设计，增强企业参与产业链协同的信心；通过安全计算、工业视觉、质量管理等复合岗位需求，推动青年技术人才服务东北制造业数字化转型。项目不以替代质检人员为目标，而是将模型定位为辅助判别工具，保留高风险结论的人工复核机制。

# 三、项目优势

## 1. 产品与服务

智检云盾由四部分组成：

（1）本地数据接入与治理模块：对接企业本地图片目录、质检台账或经批准的业务接口，完成标签校验、最小化采集、脱敏规则配置和版本管理。

（2）本地特征提取模块：调用公开、数据无关的视觉特征提取器，将质检图像转化为特征向量；私有图像不离开企业环境。

（3）安全协同训练模块：基于秘密共享组织多方训练；使用差分隐私随机梯度下降对训练过程进行保护；输出受保护的轻量缺陷识别模型。

（4）质检应用与运维模块：提供本地推理、模型版本管理、误报/漏报反馈、训练任务审计、参与方权限管理和技术支持。

项目交付分为“概念验证（POC）—私有化部署—持续订阅与模型迭代”三个层次。目标客户是有质量数据协同需求、但不愿将原始数据上传至公共云的整机厂、核心零部件企业、检测服务机构及行业数字化集成商。

## 2. 技术原理

平台遵循“本地处理、秘密共享训练、差分隐私输出、审计可追溯”的架构。

第一步，参与方在本地使用公开、数据无关的特征提取器对图像处理。第二步，特征与标签被拆分为多份秘密份额，单个参与方不能通过其所持份额恢复其他企业完整输入。第三步，各方在安全多方计算框架下训练轻量分类模型，并对单样本梯度进行裁剪、对聚合更新加入高斯噪声。第四步，输出模型由各方按约定获取并在本地部署，新图像的推理结果默认不离开工厂。

这一设计将安全训练与本地推理解耦：PEA 的直接技术参考主要用于训练阶段，云端推理、跨域身份认证和数据资产交易不被夸大为论文已解决的问题。如后续出现云端推理需求，将另行引入可信执行环境、加密推理或访问控制机制并开展评测。

## 3. 创新点

（1）面向产业链的“双层保护”：安全多方计算保护训练过程中的跨企业数据边界，差分隐私降低训练后模型泄露样本成员关系的风险。

（2）“先特征、后安全训练”的工程路径：将高开销的视觉特征提取放在企业本地，仅对低维特征训练浅层模型，以适配安全计算通信成本较高的现实条件。

（3）以质检闭环而非算法演示为产品目标：将缺陷标签治理、人工复核、模型版本、审计日志与多方权限纳入交付范围，便于企业按质量管理流程使用。

（4）面向东北装备制造的可复制场景：首期汽车零部件，后续可按“表面缺陷图像—本地特征—协同分类”的通用范式适配压缩机、泵、机床、轨道交通和电站装备部件。

## 4. 技术可行性与知识产权边界

公开论文 PEA 已在安全多方学习框架中实现安全 DPSGD，并在图像、文本分类数据集上验证了“数据无关特征提取 + 浅层模型”的效率与精度思路。该论文是本项目的技术可行性参考，而非团队论文、专利或代码成果。项目拟自主完成的工作包括：工业质检数据接口与标签规范、多方训练编排、隐私预算配置界面、模型审计功能、行业场景测试和产品化部署脚本。

项目目前未提供可证明的专利、软著或自有论文，故不在本计划书中列示。后续如形成可保护的软件模块或发明点，应在完成检索、权属确认和申报后再据实列入支撑材料。

# 四、市场分析

## 1. 行业与政策环境

制造业智能化正在从单点设备自动化走向产业链协同。质量数据涉及工艺参数、产品缺陷、供应商批次等敏感信息，企业既有提高模型效果的需求，也有保护商业秘密、履行数据安全责任的要求。数据安全法、网络安全法及与个人信息、重要数据相关的适用规则，要求企业在数据处理活动中建立分类分级、访问控制和安全管理机制。即使工业质检数据通常不以个人信息为主，仍可能包含商业秘密、客户标识和供应链敏感信息，应实行最小必要、目的限定、权限控制和可审计处理。

东北装备制造业的产业基础为项目提供了垂直验证空间。沈鼓集团覆盖大型离心压缩机、往复压缩机和离心泵的研发、设计、制造和服务；哈电集团拥有电机、锅炉、汽轮机等发电装备制造布局；中车大连覆盖机车、发动机和城市轨道车辆等产品研发制造；沈阳机床面向金属切削与数控机床制造。上述信息仅用于说明行业适配性，不作为合作背书。

## 2. 目标市场与进入路径

项目的初始服务对象不是所有制造企业，而是同时具备以下条件的客户：有一定规模的视觉质检场景；存在至少两家可协作的数据持有方；对工艺、批次和质量数据有保密要求；愿意先以小范围 POC 验证效果。

市场进入遵循“单场景、三节点、小数据、可验收”的原则。第一阶段以汽车零部件表面缺陷为切入，验证模型效果、训练时延、通信成本和数据不出域机制；第二阶段通过区域工业互联网平台、检测机构和智能制造集成商寻找客户；第三阶段复制至重大装备、轨道交通与电站装备的相似视觉质检任务。

本计划书不使用未经核验的全国市场规模或行业增速数据。项目市场空间采用自下而上的可服务收入测算：以单个客户的 POC、私有化部署和年度订阅收入为基础，结合三年目标客户数形成预测，详见第七章。该测算为经营假设，不是已签订单或市场统计结论。

## 3. 竞品与差异化

现有替代方案可分为三类：一是传统机器视觉与 AI 质检厂商，优势是现场交付经验，局限是通常服务单厂数据；二是工业互联网/云平台，优势是连接能力，局限是客户可能顾虑原始数据集中；三是通用隐私计算厂商，优势是底层安全能力，局限是缺少质检业务闭环。

智检云盾的差异化不在于宣称替代所有视觉质检厂商，而在于服务“多家企业愿意共享识别能力、但不愿共享原始数据”的细分需求。产品将把安全多方训练、差分隐私、标签治理、人工复核和审计日志组成可交付流程，形成与现场质检集成商互补的合作关系。

# 五、商业模式

## 1. 盈利模式

项目拟采取三类收入：

（1）POC 服务费：按场景收取数据梳理、原型训练、部署验证和验收服务费用；用于降低客户首次使用门槛。

（2）私有化部署费：按参与方数量、部署环境、模型类别和接口复杂度报价，包含本地节点安装、权限配置和初始模型交付。

（3）年度订阅与技术服务费：包括版本升级、模型再训练、隐私预算复核、运维支持和审计报告。对需长期迭代的客户，可按节点数或模型数续费。

报价应在完成数据复杂度、现场环境和安全要求评估后确定。本计划书中的价格仅为后续财务测算假设，不能作为对外承诺或已达成订单。

## 2. 运营模式

客户线索进入后，团队先完成场景调研与数据边界确认，签署保密和数据处理约定；再以脱敏样本或本地隔离环境开展 POC；达到双方预设的准确率、时延、误报漏报和安全审计指标后，进入私有化部署；上线后以月度运行报告、人工复核反馈和季度模型评估驱动迭代。

平台原则上不保存客户原始图像。对确需收集的日志、配置和诊断信息，实行最小化采集、加密存储、最短必要保存期限和权限审批。模型训练参与方、目的、范围、版本和隐私预算均应记录可追溯。

## 3. 营销策略

首期以“行业调研 + 联合 POC + 案例验证”获取早期客户，不以未经验证的性能数字进行宣传。渠道包括：东北地区智能制造产业园、行业协会、检测机构、工业互联网平台、设备集成商和高校成果转化活动。品牌表达聚焦“品质云盾：数据不出厂，质检可协同”，强调合规、审慎和可验收。

# 九、风险控制

| 风险类别 | 具体表现 | 应对措施 |
|---|---|---|
| 技术风险 | 差分隐私带来精度损失；多方计算训练时延较高 | 先选择图像分类等明确任务；采用公开特征提取与浅层模型；以准确率、时延、通信量和隐私预算联合验收 |
| 数据与合规风险 | 工艺秘密、供应链信息或其他敏感数据处理不当 | 数据不出厂；最小必要处理；签署数据处理约定；权限分级、日志审计、删除与退出机制 |
| 安全模型风险 | 参与方合谋、实现漏洞、模型滥用 | 明确半诚实与诚实多数假设；开展代码审查、渗透测试和密钥管理；限制模型访问频率并保留审计 |
| 市场风险 | 制造企业销售周期长、客户对新方案信任不足 | 以小范围 POC 和可量化验收切入；与检测机构、集成商合作；不承诺未经验证的性能 |
| 财务风险 | POC 收入不足、交付成本高于预期 | 分阶段付款；控制固定开支；优先复用软件模块；按项目滚动修订预算 |
| 主体与知识产权风险 | 现有主体经营范围不匹配；误用开源或论文成果 | 签约前核验主体资质；建立代码与数据来源清单；遵守开源许可证；正确引用 PEA 等公开研究 |

# 十、未来展望

## 1. 未来一年

完成三方模拟环境原型，形成数据接入、特征提取、安全协同训练、模型审计和本地推理的最小可行产品；选定一种表面缺陷任务，建立可复现实验记录；完成至少一轮行业调研和潜在客户需求访谈；根据真实团队与主体情况完善知识产权、数据处理和商业合同文件。
