# “多工厂/供应商协同质检”方案展开与标题备选

## 结论

这个方向可行，而且比把 PEA 论文直接包装成“隐私计算平台”更适合东北振兴产业升级专项赛的“数智赋能产业”组别。建议把 PEA 作为底层方法，把项目呈现为一个解决制造业真实问题的产品：**同类制造企业可联合提升缺陷识别能力，但原始质检图像、工艺参数和供应商数据不离开本地。**

PEA 的主线不是联邦学习，而是基于秘密共享的安全多方学习（MPL）与差分隐私 DPSGD。它解决了两类风险：一是训练过程中的跨主体数据泄露，二是训练完成后模型受到成员推断攻击而反推出训练样本。论文提出的“公开、数据无关的特征提取器 + 安全训练浅层模型”特别适合工业视觉质检：本地先从零件表面、焊缝、涂装等图像提取特征，再对特征进行安全协同训练，不必在多方计算中训练庞大的视觉网络。

## 项目应如何定义

### 1. 要解决的真实痛点

装备制造的缺陷样本本就稀少。单个工厂的“裂纹、划伤、气孔、焊缝缺陷、涂装缺陷、装配异常”等数据不够多，模型容易漏检；但企业通常不会把质检图像、加工参数、供应商批次等信息交给同行、客户或平台。原因包括工艺泄密、供应链责任、数据合规与竞争风险。

因此，项目的核心价值不是“把所有工厂的数据汇集起来”，而是：

> **让数据留在工厂，让多方共享缺陷识别能力。**

### 2. 可展示的技术闭环

1. **本地数据层**：每家工厂保存本地质检图像、设备/工艺参数、缺陷标签和批次信息；原始数据不上传。
2. **本地特征层**：使用公开、数据无关的视觉特征提取器，把零件图像转为特征向量。这一步在企业本地明文完成；不对私有图像进行跨厂训练或上传。
3. **安全协同训练层**：各方将特征和标签拆分为秘密份额，使用 PEA 的安全多方 DPSGD 机制训练轻量缺陷分类器。训练时各参与方都无法看到其他方的完整数据。
4. **模型保护层**：对梯度进行裁剪并加入高斯噪声，给最终模型提供差分隐私保护，降低外部用户通过查询模型推断某张图片是否参与过训练的风险。
5. **本地部署层**：将受保护的模型部署回各工厂，在本地对新图片做缺陷识别、预警和复检建议；如需云端推理，应另行设计推理阶段的安全方案，不能把 PEA 的训练保护直接等同于云端推理保护。

### 3. 与 PEA 论文的一一对应

| PEA 的内容 | 在本项目中的转化 |
|---|---|
| 秘密共享式多方学习 | 多工厂/供应商共同训练，图像和工艺数据不出厂 |
| 安全 DPSGD | 对协同训练的浅层缺陷分类模型实施梯度裁剪与差分隐私噪声 |
| 数据无关特征提取 | 使用公开视觉特征模型在本地提取零件、焊缝、表面图像特征 |
| 浅层模型训练 | 用逻辑回归或小型 MLP 做缺陷类别识别，降低安全计算开销 |
| 本地模型初始化 | 可作为可选加速模块：先在各自本地做差分隐私训练，再通过安全多方计算得到初始化模型 |

### 4. 必须如实说明的边界

- PEA 更适合 **3—5 个、相互存在合作关系的主体**；参与方很多时，安全多方计算的通信开销会迅速上升。
- 论文采用半诚实模型与“诚实多数”假设：参与方会遵守协议，但可能试图从收到的信息中推断隐私，且合谋方不能达到或超过半数。项目材料中应清楚写出该假设。
- PEA 的直接实验是图像与文本分类，工业视觉质检最贴近其证据；将其用于高频工业时序数据预测性维护需要另做原型验证，不能直接宣称论文已验证。
- 不要把项目表述为“联邦学习”。准确说法是：**基于安全多方计算与差分隐私的多主体协同学习**。本地模型初始化只是可选效率优化，不改变项目的主技术路线。

## 最推荐的首期版本

建议只做一个垂直场景：**长春汽车零部件的表面缺陷协同识别**，同时在叙事上服务东北装备制造升级。可用三个虚拟/模拟节点完成原型：零部件厂 A、零部件厂 B、独立检测机构 C。每个节点各自保留少量缺陷图像；本地提取视觉特征；在三方间安全训练一个“正常/划伤/裂纹/凹坑”等轻量分类模型；比较单厂训练、明文集中训练和 PEA 安全协同训练的准确率、训练时延、隐私预算与通信量。

这比一开始声称覆盖“全东北所有工厂”更可信，也更符合比赛对原理样机、实践过程和商业落地的考察。
